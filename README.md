# 星海绮谭 · 二次元抽卡游戏

一个纯前端、可在手机与电脑网页端游玩的二次元（美少女）抽卡（Gacha）游戏。
所有角色立绘均为 AI 绘制的**纯绿幕立绘**，由浏览器实时 **绿幕抠图（chroma-key）** 去背展示；剪影墙、图鉴、抽卡动画一应俱全，可一键部署到 **GitHub Pages**。

## 玩法

- 首页只有两个核心按钮：**抽一次** / **抽十次**（十连保底至少 1 张 SR↑）。
- 概率：**SSR 5% · SR 25% · R 70%**。
- 首页下方是 **60 位角色的剪影墙**：每抽到一位角色，对应剪影即被点亮为彩色立绘。
- **图鉴** 模式：按类型（机甲 / 校园 / 魔法 / 冒险）与获得状态筛选；点击角色查看 **人物小传** 与属性。
- 进度自动保存在浏览器 `localStorage`。

## 角色

- 共 **60** 位女性角色，覆盖 **机甲 / 校园 / 魔法 / 冒险** 四大类型，
  以及 性感 / 丰满 / 清纯 / 少女 / 英气 等多种气质。
- 品质分布：R 30 · SR 20 · SSR 10。
- 每位角色都有名字、称号、元素、属性与一段帮助玩家代入的**人物小传**。
- 当前已绘制立绘并进入抽卡池的为「已实装」角色，其余在剪影墙上显示为「COMING SOON」，
  可随时补充立绘扩展到满 60 位（见下方「扩展角色」）。

## 技术

- 纯原生 **HTML + CSS + JavaScript**，无构建步骤，开箱即用。
- `js/chroma.js`：canvas 实时绿幕抠图 + 自动裁剪人物边界，剪影用 CSS `filter` 实现。
- 完全响应式，适配手机与桌面；包含翻牌与品质光效动画。

## 本地运行

由于绿幕抠图需要读取像素（canvas 同源策略），请用本地服务器打开，而非直接双击 `index.html`：

```bash
cd 二次元抽卡游戏
python3 -m http.server 8080
# 浏览器访问 http://localhost:8080
```

## 部署到 GitHub Pages

本项目为纯静态站点，推送到仓库后在 **Settings → Pages** 选择从 `main` 分支根目录发布即可。

## 目录结构

```
二次元抽卡游戏/
├── index.html          # 页面结构
├── styles.css          # 样式 / 动画 / 响应式
├── js/
│   ├── data.js         # 60 角色数据与小传
│   ├── chroma.js       # 绿幕抠图
│   ├── gacha.js        # 抽卡概率与保底
│   └── app.js          # 主程序：剪影墙 / 图鉴 / 动画 / 详情
└── assets/
    ├── card-back.png   # 卡背（翻牌动画用）
    ├── cards/          # R / SR / SSR 品质卡片背景
    └── characters/     # 角色绿幕立绘 cNN.png
```

## 扩展角色（补满 60 位）

1. 在 `js/data.js` 中把目标角色的 `implemented` 改为 `true`。
2. 按 `assets/characters/cNN.png`（NN 为两位数 id）放入对应的**纯绿幕全身立绘**。
3. 刷新页面即可，该角色自动进入抽卡池与剪影墙点亮逻辑。
